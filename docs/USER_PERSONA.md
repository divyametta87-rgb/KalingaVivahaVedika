# User Persona & Product Goals

When AI agents are tasked with designing UI/UX flows, generating copy, or building feature logic, they must align their outputs with the target user defined below.

## 🎯 Target Audience
**Primary User:** Young Indian professionals (24-35 years old) AND their parents (50-65 years old) seeking matrimonial matches within the Kalinga community.

## 📱 User Expectations & Pain Points
1. **Trust is #1:** Users are sharing highly sensitive personal and family data. The UI must feel secure, premium, and trustworthy. Errors, broken links, or generic "bootstrap" designs diminish trust immediately.
2. **Ease of Use for Older Generations:** Navigation must be extremely intuitive. Fonts should be readable, and calls to action (CTAs) must be obvious.
3. **Mobile First:** The vast majority of traffic will come from mobile devices (Android/iOS). Desktop layouts are secondary but still required.
4. **Cultural Relevance:** The platform should gracefully handle specific search criteria like Education, Profession, Location, and Sub-Caste nuances without feeling exclusionary or overwhelming.

## ✨ The "WOW" Factor Guidelines
When creating the frontend, agents must ensure:
- **Color Palette:** Use elegant, culturally fitting colors (deep reds, golds, clean whites) without being overly bright or tacky.
- **Micro-Animations:** Buttons should have hover effects. Route transitions should be smooth. Loading skeletons should be used instead of jarring spinners whenever possible.
- **Empty States:** When a user has 0 matches or 0 messages, show a beautiful, encouraging illustration or message—do not show a blank screen.
